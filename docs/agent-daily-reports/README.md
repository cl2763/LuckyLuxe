# Lucky Luxe Agent Diaries

This folder stores long-running diaries for the Lucky Luxe multi-agent workflow.

The report structure is organized by Agent, not by date. Each Agent owns one file and appends timestamped entries over time, so opening that file shows the Agent's full history.

## Files

- `product-strategy-agent.md`
- `backend-api-agent.md`
- `frontend-web-agent.md`
- `mini-program-agent.md`
- `ai-customer-service-agent.md`
- `qa-testing-agent.md`
- `deployment-ops-agent.md`

## Entry Format

Each update should append a timestamped entry with these sections:

- Problems found
- Fixes / decisions made
- Final effect
- Verification evidence
- Open blockers / risks
- Next steps

The most important content is what problem was found, how it was solved, and what effect was finally achieved.
